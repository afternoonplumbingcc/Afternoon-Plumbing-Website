## Form Flow

### Contact Form (Simple)
```
User sees ContactForm
   ↓
Fill in fields (name, phone, service, message)
   ↓
Client-side validation
   ↓
Submit to Netlify (POST /)
   ↓
Success: Show thank you message
   ↓
Option: Submit another request
```

### Quote Form (Multi-Step)
```
User sees QuoteForm (Step 1)
   ↓
Select Service Category → Select Specific Service
   ↓
Validation → Continue
   ↓
Step 2: Enter Address (street, city, zip)
   ↓
Validation → Continue
   ↓
Step 3: Contact Info (name, phone, email)
   ↓
Validation → Continue
   ↓
Step 4: Project Details (description, urgency)
   ↓
Validation → Submit to Netlify
   ↓
Success: Show confirmation + reference number
```

## Conversion Paths

1. **Hero CTA** → QuoteForm (high intent)
2. **Services Grid** → QuoteForm (interest)
3. **Emergency Banner** → WhatsApp/Call (urgent)
4. **Contact Form** (alternate for quick questions)
5. **Mobile Bar** → Call/WhatsApp/Form (mobile users)
6. **Footer** → Contact/Call
7. **Review Section** → Trust → Contact
8. **SocialContact Widget** → Direct message/call

## Data Flow

```
Form Input → Component State
   ↓
Validation
   ↓
FormData object
   ↓
FormData → FormData (Netlify format)
   ↓
Fetch POST → / (Netlify)
   ↓
Netlify receives → Email notification
   ↓
Netlify dashboard shows submission
```

## State Management

- Each form component: `useState` for form fields, submission status, errors
- No external state management (Context/Redux)
- Prop drilling minimal (mostly self-contained forms)

## Styling Strategy

- Tailwind CSS v4 for utility classes
- Custom CSS in `App.css` for:
  - Custom color palette (amber/red)
  - Component-specific styles
  - Responsive breakpoints
- Framer Motion for:
  - Form transitions (slide, fade)
  - Button hover/tap animations
  - EmergencyCTA entrance/exit
  - Pulse animations (emergency badge)

## SEO Implementation

- `index.html`: Meta tags, Open Graph, Twitter Cards
- `GBPSchema.tsx`: LocalBusiness + Service JSON-LD (auto-injected)
- `ReviewSchema.tsx`: AggregateRating + Reviews JSON-LD
- Semantic HTML: `<section>`, `<header>`, `<nav>`, `<footer>`
- Heading hierarchy: H1 (hero) → H2 (sections) → H3/H4
- Alt text on images (if any)
- Sitemap.xml (can be auto-generated by Netlify)

## Performance

- Vite build with Rollup bundling
- Tree-shaking: unused exports eliminated
- CSS extracted to separate file (caching)
- No external font requests (system fonts)
- Images: Use SVG icons (lucide-react) → minimal asset load
- Total JS: ~110 KB gzipped (fast load)

## Accessibility

- Labels for all form inputs
- Keyboard navigation support
- Focus states on interactive elements
- ARIA attributes where needed
- Color contrast: amber on white, white on red/amber
- Error messages announced to screen readers

## Browser Support

- Modern browsers (Chrome, Firefox, Safari, Edge)
- ES2023 target (basic ES6 features)
- CSS Grid + Flexbox (responsive)
- No polyfills for old browsers (IE11 not supported)

## Mobile Responsive

- Mobile-first approach
- Breakpoints: sm (640px), md (768px), lg (1024px)
- MobileContactBar appears only on mobile
- Form fields stack vertically on mobile
- EmergencyCTA condenses on mobile
- Navigation hidden on mobile (could add hamburger in future)

---

**All components are modular, reusable, and production-ready.**
